# Groomsta Internship Handbook – 2025

### Complete Guide for All Engineering Interns (Frontend, Backend, Core Systems)

## 1. Welcome to Groomsta

Welcome to the engineering internship program at **Groomsta** — a premium home & in-salon grooming
service platform for men and women across Delhi NCR.

This handbook contains **everything you need to know** during your internship: expectations, workflows,
meetings, code practices, communication, project structure, and deadlines.

Your work directly impacts our live product and end users. We expect professionalism, commitment, and
clarity.

**Our Motto:** _Move fast, build well, communicate clearly._

## 2. Internship Duration & Goals

### Duration: 6–8 weeks (extendable based on performance)

### Primary Goal: Deliver a production-ready MVP of Groomsta, including:

```
Customer booking platform (web)
Partner dashboard (barbers/salons)
Core backend systems
Authentication, KYC, payments
Admin controls
```
You will work in a structured team environment and follow industry-standard engineering practices.

## 3. Team Structure & Individual Roles

Each intern has a well-defined responsibility based on specialization.

### Dev 1 — Core Backend + Security + Payments

Responsible for: - Authentication system (login, signup, JWT, refresh tokens) - KYC: photo + ID upload -
Payment integration (Razorpay) - Database schema (PostgreSQL / Supabase) - Security protocols & best
practices - API architecture & documentation

## 4. Work Philosophy & Culture

At Groomsta: - **Communication > Assumptions** - **Clean code > Fast code** - **Teamwork > Individual glory** -
**Consistency > Intensity**

You are expected to take ownership, ask questions, clarify doubts, and deliver your tasks on time.

## 5. Weekly Workflow Structure

### A. Weekly Team Meetings

**Every Monday, 9 PM – 10 PM (Google Meet)** Agenda: 1. Last week’s progress review 2. Upcoming tasks 3.
Roadblocks discussion 4. Aligning across frontend, backend, and core systems

### B. Mid-Week Sync

**Every Thursday, 9:30 PM (15–20 mins)** - Quick check-in - Make sure no one is stuck

### C. End‑of‑Week Demo (Friday or Saturday)

```
Each intern must show live progress
Feature demo or backend tests
```
## 6. Daily Workflow Expectations

### 1. Daily Standup (Message Only — WhatsApp/Slack)

Before 11 AM, send: - **Yesterday:** What did you complete? - **Today:** What will you do today? - **Blocked:**
Anything blocking you?

### 2. Daily Coding Hours

Interns are expected to dedicate **2–4 hours/day** (flexible timing). No micro‑tracking — focus on output.

### 3. Commit Expectations

```
Minimum 1–2 meaningful commits per day
Use proper commit messages:
feat: added customer booking UI
fix: payment callback validation bug
chore: updated README
```
## 7. Code Workflow & Git Guidelines

### Branch Naming Rules

```
feature/auth-system
feature/customer-booking-ui
feature/partner-dashboard
fix/payment-webhook
hotfix/ui-bug
```
### Never push directly to main

Every feature → **PR → Review → Merge**

### Pull Request (PR) Rules

Every PR must include: - What you built - Screenshots (for UI) - API test results (for backend) - What needs to
be reviewed

### Review Requirements

```
Each PR must be reviewed by at least 1 teammate
No self‑merge allowed
No untested code in main
```

## 11. Task Assignment Process

Tasks will be created as **GitHub Issues** with: - Title - Description - ACs (Acceptance Criteria) - Labels
(frontend, backend, core, urgent) - Assigned developer - Deadline

Each intern must: - Pick tasks only assigned to them - Keep issue status updated (To‑Do → In‑Progress →
Review → Done)

You will be judged based on: - Consistency - Communication - Quality of code - Ability to work
independently - Ownership - Meeting deadlines - Teamwork

## 15. Internship Completion Requirements

To successfully complete the internship: - Attend all major meetings - Submit all assigned tasks -
Demonstrate weekly progress - Submit final report of your work (screenshots + PR links) - Participate in final
demo

## 16. Important Rules

```
No copying code from random sources without understanding
No unlicensed assets or copyrighted UI
No sharing private repository code outside Groomsta
Maintain clean communication at all times
```
## 17. What You Should Do If Stuck

```
Try searching the problem for 30 mins
Check documentation
Check related PRs / issues
Ask teammates
If still stuck → direct message the founder
```
We encourage independence but support you fully when needed.
